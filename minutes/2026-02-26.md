### Libraries and Mints
* CDK
    * [v0.15.1]( https://github.com/cashubtc/cdk/releases/tag/v0.15.1) 
    Add limits to secret and witness, build_with_seed with custom paths 
    * [v0.15.0](https://github.com/cashubtc/cdk/releases/tag/v0.15.0)
        - Wallet Saga pattern — all wallet operations (mint, melt, send, receive, swap) now have robust crash recovery. New two-phase melt flow with prepare/confirm pattern.
        - Adds NUT-26 (Payment Request Bech32m Encoding) for BIP-321/BIP-353 compatibility,  async wallet operations support, multiMintWallet removed in favor of WalletRepository, authentication (NUT-21/22) now always enabled, keyset V2 is the new default
* CDK Bindings
    * [cdk-swift 0.15.1](https://github.com/cashubtc/cdk-swift/releases/tag/v0.15.1)
    * [cdk-kotlin 0.15.1](https://github.com/cashubtc/cdk-kotlin/releases/tag/v0.15.1)
* cashu-ts
    * [v3.5.0](https://github.com/cashubtc/cashu-ts/releases/tag/v3.5.0)
        - Async Melt via POST body (not headers), P2BK (Pay-to-Blinded-Key) is now stable, Removed interim sig_all message format
    * [v3.4.1](https://github.com/cashubtc/cashu-ts/releases/tag/v3.4.1)
    * [v3.4.0](https://github.com/cashubtc/cashu-ts/releases/tag/v3.4.0)

        - Prerelease 
* Nutshell
    * [v 0.19.2](https://github.com/cashubtc/nutshell/releases/tag/0.19.2)
        - NUT-18 Payment Request support (core + CLI), fix: no witness if no spending condition
    * [v0.19.0](https://github.com/cashubtc/nutshell/releases/tag/0.19.0)
        - Adds Lightning address support via npubx.cash. Mint now supports background sync for unpaid invoices and built-in Tor service. New P2PK spending rules (both spend paths possible after timelock). Keyset metadata now included in keys response. Multiple new contributors joined the project.
